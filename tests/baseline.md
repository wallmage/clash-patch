# 当前测试基线

本文件记录现行测试范围，不定义产品功能。产品要求见 `docs/superpowers/specs/2026-07-20-clash-patch-skill-design.md` 和 `clash-patch/references/patch-policy.md`。

## 必须通过

- Ruby：macOS 三档用途、全部订阅共享的国内域名规则提供器、名称冲突保护、普通安装整批预检与失败零写入、预检快照与事务提交逐份绑定、单次刷新后重新预检、连续刷新保留最新字节、安全更新提交前刷新清理未提交事务并允许后续重试、备份/比较/恢复且恢复写入前持全局操作锁并处理未完成事务、安全更新事务、远程订阅清单严格结构、HTTPS、文件唯一映射、curl 配置注入防护、UTF-8、二次转换与 Mihomo 失败关闭；订阅自动更新分阶段所有权记录与安全恢复、带截止时间的文件锁、同目录原子交换、提交后身份复核失败恢复、批量更新期间的路径与文件身份变化、中途写入失败恢复、同一 inode 别名拒绝、空配置目录拒绝、运行刷新、恢复当前订阅时逐项复核缓存刷新、TUN、代理选择、国内与境外 DNS 和连通性，任一步失败都拒绝生效；运行回退已恢复当前文件时不做第二次候选身份回退，继续恢复其余订阅并单独报告运行内核仍未恢复；候选提交后只有全部原字节才能删除事务，回退期间的外部刷新保持不动并保留恢复记录；候选交换后即使提交身份尚未登记也按临时候选身份和字节恢复，相同候选字节但不同身份的外部刷新保持不动；加载候选后被强制结束时，下次安全更新在下载前恢复文件并重新加载原运行配置；文件与内核恢复状态分别上报、YAML 1.2、Mihomo 校验，以及同时绑定源端口、TCP、新连接 ID、域名边界并拒绝非代理终点的分流验证。
- 双平台 conformance：`tests/fixtures/main_group_cases.json` 由 macOS Ruby 与 Windows JavaScript 共用，覆盖 Mihomo 的全部代理组类型、只有 AI 分组和纯直连组的最终保底；每个案例用人工确认后的 `expected_config_sha256` 锁定完整结果，两端分别匹配后再做完整输出深比较，发生变化的案例再次执行后必须不再变化。
- 覆盖率门槛：macOS 每个 Ruby 生产模块及其合计都必须达到 100% 行覆盖，全部模块的分支合计不得低于 75%；真实 Ruby 命令入口与缺依赖安装包在子进程中执行并断言公开输出，其 Coverage 按源码 SHA-256 合并回对应生产文件；Windows JavaScript 转换引擎必须达到 100% 行、100% 函数且分支不低于 80%。覆盖统计跟随拆分后的全部生产模块，并列出未覆盖行和未覆盖分支所在行。门槛只能上调；不得通过忽略正常生产行、无断言调用或删除防御代码凑数字。
- Shell/CMD 包装器：帮助、参数与退出状态、档位保存/复用、普通失败恢复旧档位、强杀后把新档位保留为下一次 Patch/安全更新的恢复意图、档位 3 降档前强制安全卸载、能力预检、卸载和备份保留等公开行为；macOS 还验证安装失败和安全卸载都按所有权状态恢复自动更新，文件卸载与自动更新恢复属于同一可恢复事务，失败时必须同时保留用途档位、关闭自动更新并保留所有权，并以真实 Shell → 真实 Ruby 主程序 → 受控 Mihomo 可执行文件验证公开安装成功路径，不拿包装器逐行覆盖率冒充安全性。
- macOS 包装流程互斥：普通安装、安全更新和安全卸载共用跨进程操作锁，覆盖档位读取、自动更新所有权、订阅事务、卸载最终核对与提交。生产探针把卸载停在最终核对后、路径删除前；并发公开安装必须返回 `operation_in_progress`，卸载完成后重试才能发布新档位。另一个探针在 `READY` 后删除完成、自动更新恢复前强杀整个卸载进程组；下一次公开安装必须先完成安全卸载并验证收据，再写入新档位。分别移除安装端、卸载端的锁获取或安装端的未完成卸载恢复时，mutation 必须稳定失败。
- macOS 卸载强杀恢复：READY 前的不完整暂存必须直接丢弃且不得改动安装文件；READY 后恢复文件只能原子发布，恢复再次被强杀后下一次运行必须还原完整字节。对应 mutation 分别移除 READY 边界和退回直接复制，测试都必须失败。
- macOS 卸载配置事务交接：公开 Patch 在候选写入和运行加载后被强杀，随后公开卸载必须先恢复原文件与原运行配置，再删除安装状态或恢复订阅自动更新。恢复失败时全部受保护状态保持原样；移除卸载前恢复调用的 mutation 必须失败。
- macOS 普通 Patch 运行恢复：故障注入在当前订阅加载候选、事务尚未完成时强杀进程；下一次运行必须恢复原文件和原运行配置，忽略只存在于候选中的临时选择器。强杀恢复或本次自动刷新回退被控制器拒绝时，都必须保留事务，并在新候选校验前停止；`--no-reload` 只能恢复文件和保留事务，不得校验新候选。对应 mutation 移除运行恢复、让 `--no-reload` 绕过恢复或提前删除事务时，测试都必须失败。
- macOS 安全更新运行恢复：旧事务恢复必须从存储目录全部顶层配置识别活动项；活动 `config.yaml` 不在远程订阅目标时仍要通过控制器重新加载。旧事务或本次安全更新回滚的运行检查失败时保留事务，并在下载或新写入前停止。故障注入断言控制器拒绝时记录仍存在；mutation 移除共享恢复入口或强制删除待恢复记录时必须失败。
- macOS 备份恢复事务：旧 Patch 事务必须在备份哈希判断前恢复文件与运行配置，控制器拒绝时保留记录且不调用备份校验。备份替换后、当前订阅运行检查前的强杀由下一次 Patch 恢复；正常运行检查期间事务仍存在，文件已经与备份相同时也要留下可重试记录，运行回退未完成时继续保留。对应 mutation 移除备份事务、跳过相同文件事务、提前提交或在运行恢复失败时删除记录，测试都必须失败。
- macOS 生产故障探针：独立 CI job 通过单一可执行入口在 Ruby 3.3 与系统 Ruby 下收集四组结果，任一失败都让 job 失败且不会遮住其余组；探针通过真实文件替换、精确同步闸门与 SIGKILL 检查提交后登记失败、普通批量和安全更新强杀恢复、同 inode 别名、校验调用方消失后的子进程与临时文件、卸载 staging 后目标被替换、卸载最终核对后并发安装被操作锁拒绝，以及删除完成后强杀的卸载由下一次安装先完成恢复；普通回归默认跳过这些已确认会失败的探针，契约行为测试和 mutation 同时锁定强制启用、完整执行与失败传播。
- Mihomo 集成：macOS CI 分别校验官方 v1.19.27 和 v1.19.29 的 arm64/amd64 压缩包 SHA-256，再用四个真实内核验证原始配置和档位 1、2、3 的补丁产物；独立 runner 必须确认每次恰好执行一个、断言非零且没有跳过的真实内核案例，并验收绑定内核 SHA-256、随机 nonce、三个档位及原始/补丁六次校验的完成凭据，测试名称漂移、零测试、提前返回或档位循环缩短都不能成功。Windows CI 校验同版本官方 amd64 zip 的 SHA-256，并让两个真实内核在 PowerShell 5.1 与 7 下分别通过公开安装入口、三档基础配置和完整订阅的 `Script.js` 转换产物复验；Windows 各档失败分别记录后统一报错，前一档不能遮住后两档；缺少指定内核时测试直接失败。
- 发布包：macOS 按公开分发清单生成 tar 包，在带空格和中文的独立目录解压，再运行 Shell/Ruby 入口、Windows JavaScript 语法检查、Ruby 档位 1 演练，以及公开 Shell → 系统 Ruby → 受控 Mihomo 的真实档位 1 安装；Windows 从 zip 解压目录完成真实档位 1 安装，并注入缺模块包验证零改动失败。依赖路径 mutation 必须让 macOS 发布包测试失败。
- Node.js：Windows 全局扩展脚本与 macOS 策略一致；档位 1、2 只应用共享国内域名基线，档位 3 再应用完整增强。
- PowerShell 5.1 与 7：两个独立 CI job 运行完整套件，避免一个版本失败后跳过另一个版本；父进程和实际子进程都必须自证 PSEdition 与主版本，完整套件只有执行到全部断言和清理之后才生成绑定运行时的完成凭据，CI 必须独立验收凭据；覆盖 Windows 三档用途、安装/卸载、失败时保持原档位、安全卸载后的真实降档、默认 AppHome 多候选时拒绝歧义、远程订阅自动更新的原值与原 `option` 形态所有权、重复安装不改写原状态、受管脚本后缀和已有脚本的唯一入口声明之外都不得重新绑定 `main`、严格状态结构与单次 snapshot、运行客户端且存在应用状态时整批零改动、写删重叠和重复目标预检、新建文件的空原内容必须可写入事务日志、全目标与父目录句柄先取得、文件身份与原字节双重核对、阻断读取/写入/改名/替换、句柄绑定删除、删除标记失败恢复、目录/重解析点/硬链接拒绝、父目录联接点竞态拒绝、坏备份先拒绝、内部状态恢复禁令、备份完整同步并设置私有权限后才原子发布正式名称且发布前强杀不可见、AppHome 的大小写、扩展路径和 SUBST 别名与改名不可绕过互斥，系统提供 8.3 短路径时还要验证备份身份一致；相对目录备份身份在路径别名和目录改名后保持一致、公共安装被强杀后由公共卸载恢复、公共卸载恢复再次被强杀后仍可继续、公共卸载被强杀后由公共安装恢复、公共备份恢复写到一半被强杀时不得留下截断文件、进程强杀后的批量写入与删除事务恢复、中断新建文件只删除本工具完整候选或候选前缀、恢复中的原始字节前缀可以续写但其他内容与身份替换必须拒绝、损坏事务日志严格拒绝、失败恢复不覆盖并发内容、Mihomo 超时终止全部后代进程、候选配置公开前先启动调用方死亡清理监视且调用进程消失后不遗留候选配置，候选配置与事务日志都只允许当前用户和系统管理员读取、嵌套 `option` 层级、远程订阅清单严格结构与严格 UTF-8、安全更新校验字节绑定、受管全局脚本与当前安装包及保存档位一致、空代理组整批恢复、合法块状与行内自动代理组及只有 AI 分组的配置都可继续由 Mihomo 校验，以及从当前运行规则的最后一个 `MATCH` 绑定任意名称主代理组，同时绑定源端口、TCP、新连接 ID、域名边界、非代理终点拒绝和每次观察后清理挂起 curl 进程的分流验证；测试失败诊断只记录输出长度与 SHA-256，不回显子进程原文；全库通过 PowerShell AST 禁止在赋值、参数、循环、自增减或变量命令中写入只读自动变量。
- Windows 卸载运行状态提交条件：生产故障注入把公开卸载停在全部目标句柄和原字节已经核对、事务记录尚未发布的位置，再启动真实名称为 `clash-verge.exe` 的进程。卸载必须返回 `partial`，全部受保护文件和恢复状态保持原字节，事务准备记录被清理；移除提交条件参数的 mutation 必须失败，客户端停止后的重试必须完成恢复。
- Windows 安装、备份恢复和卸载运行状态提交条件：档位 3 安装与公开备份恢复复用同一锁定目标边界故障注入，在事务核对全部句柄和原字节后启动真实名称为 `clash-verge.exe` 的进程；两个流程都必须在写入前失败，受保护目标保持原字节且不留下事务记录，客户端停止后重试成功。移除任一提交条件或丢弃单写事务条件的 mutation 必须失败。
- Windows 卸载与尚未验收的安全更新：`-SnapshotProfiles` 创建清单后立即调用公开卸载，必须返回 `partial/safe_update_pending`，完整 AppHome 内容逐字节不变，随后 `-VerifySafeUpdate` 仍可完成验收或恢复；移除清单检查的 mutation 必须失败。
- Windows 较短内容替换中断：公开备份恢复把较短备份写回较长当前文件，并在完整写入后、旧实现事后截断前强制结束进程；下一次公开操作必须按事务记录恢复精确原字节。故障注入必须能让“先写后截断”的旧顺序失败，证明测试会发现新内容与旧尾部拼接后无法恢复的状态。
- Windows 新建目标准备事务：公开首次档位 1 安装在全部新目标已创建、主事务记录尚未发布时强制结束进程；必须留下私有准备记录，下一次公开安装先清理仍为空的新目标，再生成有效用途状态并成功完成。另一个探针在主事务记录已发布、准备记录尚未删除时强杀；下次公开安装先按主记录删除新目标，再允许准备记录中的目标已经不存在。对应 mutation 拒绝该缺失目标时必须失败。
- Windows 安全更新失败恢复：订阅恢复与验收清单删除使用同一个写删事务。生产探针在恢复 helper 返回后强杀公开验收进程；下一次公开验收不得把已经恢复的旧订阅报告为本次更新成功。并发订阅变化、坏备份或事务失败时必须保留清单且整批不写。
- 合同测试：公开文件、Patch/Diagnostics 的“取证 → 修复 → 复测 → 失败恢复并继续迭代”完成闭环、AdGuard 兼容路径及其 Fake-IP 重用时按域名接入 Mihomo 的出站代理修复、配置历史、安全更新、Sub Agent 边界、Windows Computer Use 条件、双平台分流验证、策略生成、平台边界和 CI 配置；全部公开命令的 JSON v1 必须保持单对象输出、退出码一致、稳定 `code`/`operation`、规定字段类型和脱敏边界，默认中文输出不变。
- 语法与格式：Ruby、JavaScript、Shell、PowerShell、全部 PowerShell 文件的严格 UTF-8 BOM、策略同步和 `git diff --check`。
- 测试机制：CI 的 macOS 与 Windows job 都有总超时；独立生产故障探针先记录失败并继续执行，最后统一让 job 失败，避免前一个问题遮住后续测试。GitHub Actions 的 `shell` 字段必须使用静态受支持值，契约和 mutation 必须在推送前拦截会让 workflow 零 job 失败的动态 shell。mutation smoke test 覆盖文件写入恢复、安全更新回退、路径身份、整批预检、自动更新补偿、默认 Mihomo 解析、结果档位边界、分流流量绑定、发布包依赖与公开安装实际写入、生产探针 CI 开关、Windows 故障汇总和二次转换保护。mutation 的语法错误、加载失败或超时不能冒充被测试发现。
- Windows 真实 Mihomo：每个 PowerShell 版本和 Mihomo 版本的矩阵任务必须生成绑定当前随机 nonce、PowerShell 运行时、内核数量及档位 1、2、3 完成记录的凭据；只看到子进程退出码不能算完成。

## 必须持续防止

- 退出、停止或重启 Clash。
- macOS 自动监听、自动重载、自动切换 TUN 或节点。
- 修改未启用的存储位置、旧 iCloud 容器、备份或废弃订阅。
- 用固定境外 DNS 覆盖节点启动解析。
- 自动创建额外安全代理组或替用户选择 AI 节点。
- 在第一、二档执行第三档泄漏检查、TUN/IPv6、WebRTC 或 AI 分组增强；共享国内域名基线不属于第三档增强。
- 把 Windows Computer Use 当成后台或锁屏能力，或在当前会话没有工具时假装完成界面验收。
- 让 Sub Agent 并行写配置、更新/恢复订阅、抢同一界面或同时制造测试流量。
- 并发刷新覆盖、重复代理组、非幂等转换、无限等待和泄露敏感信息。

具体命令和 CI 流程见 `AGENTS.md`。
