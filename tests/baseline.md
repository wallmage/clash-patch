# 当前测试基线

本文件记录现行测试范围，不定义产品功能。产品要求见 `docs/superpowers/specs/2026-07-20-clash-patch-skill-design.md` 和 `clash-patch/references/patch-policy.md`。

## 必须通过

- Ruby：macOS 三档用途、全部订阅共享的国内域名规则提供器、名称冲突保护、普通安装整批预检与失败零写入、备份/比较/恢复、安全更新事务、订阅自动更新分阶段所有权记录与安全恢复、带截止时间的文件锁、同目录原子交换、提交后身份复核失败恢复、批量更新期间的路径与文件身份变化、中途写入失败恢复、同一 inode 别名拒绝、空配置目录拒绝、运行刷新、恢复当前订阅时重新清缓存并复核 TUN、代理选择、DNS 与连通性、文件与内核恢复状态分别上报、YAML 1.2、Mihomo 校验，以及同时绑定源端口、TCP、新连接 ID、域名边界并拒绝非代理终点的分流验证。
- 双平台 conformance：`tests/fixtures/main_group_cases.json` 由 macOS Ruby 与 Windows JavaScript 共用；每个案例用人工确认后的 `expected_config_sha256` 锁定完整结果，两端分别匹配后再做完整输出深比较，发生变化的案例再次执行后必须不再变化。
- 覆盖率门槛：macOS 每个 Ruby 生产模块及其合计都必须达到 100% 行覆盖，全部模块的分支合计不得低于 75%；Windows JavaScript 转换引擎必须达到 100% 行、100% 函数且分支不低于 80%。覆盖统计跟随拆分后的全部生产模块，并列出未覆盖行和未覆盖分支所在行。门槛只能上调；不得通过忽略正常生产行、无断言调用或删除防御代码凑数字。
- Shell/CMD 包装器：帮助、参数与退出状态、档位保存/复用、失败时保持原档位、档位 3 降档前强制安全卸载、能力预检、卸载和备份保留等公开行为；macOS 还验证安装失败和安全卸载都按所有权状态恢复自动更新，失败时保留用途档位，并以真实 Shell → 真实 Ruby 主程序 → 受控 Mihomo 可执行文件验证公开安装成功路径，不拿包装器逐行覆盖率冒充安全性。
- Mihomo 集成：macOS CI 分别校验官方 v1.19.27 和 v1.19.29 的 arm64/amd64 压缩包 SHA-256，再用四个真实内核验证原始配置和档位 1、2、3 的补丁产物；缺少指定内核时测试直接失败。
- Node.js：Windows 全局扩展脚本与 macOS 策略一致；档位 1、2 只应用共享国内域名基线，档位 3 再应用完整增强。
- PowerShell 5.1 与 7：两个 CI 入口都必须由父进程和实际子进程自证 PSEdition 与主版本；覆盖 Windows 三档用途、安装/卸载、失败时保持原档位、安全卸载后的真实降档、远程订阅自动更新的原值与原 `option` 形态所有权、重复安装不改写原状态、严格状态结构与单次 snapshot、运行客户端且存在应用状态时整批零改动、写删重叠和重复目标预检、全目标与父目录句柄先取得、文件身份与原字节双重核对、阻断读取/写入/改名/替换、句柄绑定删除、删除标记失败恢复、目录/重解析点/硬链接拒绝、父目录联接点竞态拒绝、坏备份先拒绝、内部状态恢复禁令、AppHome 路径别名与改名不可绕过互斥、相对目录备份身份在路径别名和目录改名后保持一致、进程强杀后的批量写入与删除事务恢复、失败恢复不覆盖并发内容、嵌套 `option` 层级、远程订阅清单、安全更新校验字节绑定，以及同时绑定源端口、TCP、新连接 ID 与域名边界的分流验证；全库通过 PowerShell AST 禁止在赋值、参数、循环、自增减或变量命令中写入只读自动变量。
- 合同测试：公开文件、Patch/Diagnostics、AdGuard 兼容路径、配置历史、安全更新、Sub Agent 边界、Windows Computer Use 条件、双平台分流验证、策略生成、平台边界和 CI 配置；全部公开命令的 JSON v1 必须保持单对象输出、退出码一致、稳定 `code`/`operation`、规定字段类型和脱敏边界，默认中文输出不变。
- 语法与格式：Ruby、JavaScript、Shell、PowerShell、全部 PowerShell 文件的严格 UTF-8 BOM、策略同步和 `git diff --check`。
- 测试机制：CI 的 macOS 与 Windows job 都有总超时；mutation smoke test 覆盖文件写入恢复、安全更新回退、路径身份、整批预检、自动更新补偿、分流流量绑定和 Windows 二次转换保护。mutation 的语法错误、加载失败或超时不能冒充被测试发现。

## 必须持续防止

- 退出、停止或重启 Clash。
- macOS 自动监听、自动重载、自动切换 TUN 或节点。
- 修改未启用的存储位置、旧 iCloud 容器、备份或废弃订阅。
- 用固定境外 DNS 覆盖节点启动解析。
- 自动创建额外安全代理组或替用户选择 AI 节点。
- 在第一、二档执行第三档泄漏检查、TUN/IPv6、WebRTC 或 AI 分组增强；共享国内域名基线不属于第三档增强。
- 把 Windows Computer Use 当成后台或锁屏能力，或在当前会话没有工具时假装完成界面验收。
- 让 Sub Agent 并行写配置、更新/恢复订阅、抢同一界面或同时制造测试流量。
- 并发刷新覆盖、重复代理组、非幂等转换、无限等待和泄露敏感信息。

具体命令和 CI 流程见 `AGENTS.md`。
